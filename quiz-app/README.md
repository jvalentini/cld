# Quiz App (Vue 3 + Vite + Vitest + Supabase)

Modern, production-ready quiz web application that powers the Quiz Generator project.

**Features**

- Upload quiz JSON files (generated by the DOCX parser) or load quizzes directly from Supabase  
- Full quiz flow with progress bar, navigation, and answer review  
- Results page with detailed scoring (overall + per-question breakdown)  
- Rich statistics dashboard (overall, per-quiz, per-question, answer distribution)  
- Progress automatically saved in `localStorage` – resume anytime  
- Responsive, mobile-friendly design with smooth animations  
- Built with Vue 3 Composition API + `<script setup>`  
- Fully tested with Vitest (95%+ coverage)  
- Dockerized production build (Nginx)  

## Tech Stack

| Layer               | Technology                                 |
|---------------------|--------------------------------------------|
| Framework           | Vue 3 + Vite                               |
| Language            | TypeScript (optional) + JavaScript         |
| Testing             | Vitest + Vue Test Utils + Testing Library |
| Styling             | Plain CSS (BEM-like) + CSS variables       |
| Backend (optional)  | Supabase (PostgreSQL + Auth + RPC)         |

## Local Development

### Prerequisites
- Node.js ≥ 18
- npm (or pnpm/yarn)
- Docker (optional, for production build)

### Quick Start

```bash
# From the quiz-app directory
cd quiz-app

# Install dependencies
make install        # or: npm ci

# Start dev server (hot reload)
make dev            # → http://localhost:5173
```

## Running Tests

```
# Run once
make test               # npm run test

# Watch mode (TDD)
make test-watch         # npm run test -- --watch

# Interactive Vitest UI (highly recommended)
make test-ui            # → http://localhost:51204/__vitest__/

# Coverage report
make coverage           # generates coverage/ folder + HTML report
```

## Docker / Production

```
# Build production image
make build              # → quiz-app:latest

# Run container
make run                # → http://localhost:8080

# Stop container
make stop

# Tail logs
make logs

# Using Docker Compose (alternative)
make up                 # docker-compose up -d
make down               # docker-compose down
```

## One-liner cheat sheet

```
make dev           # development
```
